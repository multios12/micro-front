# 管理ページ詳細設計書

## 1. 概要
本書は、`1.site-design.md` および `2-1.detail-common.md` をもとに、管理ページの詳細設計を整理するための文書とする。

## 2. 対象範囲
- 管理API詳細
- 管理ページレイアウト
- 管理ページコンポーネント設計
- 管理ページ用ディレクトリ構成

## 3. 管理API設計
### 3.1 サイト情報取得API
- 対象API: `GET /admin/api/site`
- レスポンス: `2-1.detail-common.md` の `5.2.1 site_settings` 準拠
- 変換項目:
  - `tabs`: JSON文字列ではなく配列で返却する
  - `site_description`: Markdown文字列として返却する

### 3.2 サイト情報更新API
- 対象API: `PUT /admin/api/site`
- リクエスト: `2-1.detail-common.md` の `5.2.1 site_settings` 準拠
- 除外項目: `id`, `updated_at`
- リクエスト時の変換項目:
  - `tabs`: 配列で受け取り、DB保存時にJSON文字列へ変換する
  - `site_description`: Markdown文字列として保存する
- レスポンス: `2-1.detail-common.md` の `5.2.1 site_settings` 準拠
- レスポンス時の変換項目:
  - `tabs`: JSON文字列ではなく配列で返却する
  - `site_description`: Markdown文字列として返却する

### 3.3 記事一覧取得API
- 対象API: `GET /admin/api/blogs`
- 入力条件:
  - `page`: 取得ページ番号。デフォルトは `1`
  - `per_page`: 1ページあたりの件数。デフォルトは公開側と同様に `20`
  - `status`: 公開状態。`public` / `private`
- レスポンス:
  - `items`: `2-1.detail-common.md` の `5.2.2 blogs` の配列
  - 除外項目: `content`
  - `items[].summary`: 概要
  - `page`: 現在ページ
  - `per_page`: 1ページあたり件数
  - `total`: 総件数
  - `total_pages`: 総ページ数
- 変換項目:
  - `items[].category`: 文字列で返却する

### 3.4 記事詳細取得API
- 対象API: `GET /admin/api/blogs/{blog_id}`
- レスポンス: `2-1.detail-common.md` の `5.2.2 blogs` 準拠
- 変換項目:
  - `category`: 文字列で返却する

### 3.5 記事新規作成API
- 対象API: `POST /admin/api/blogs`
- リクエスト: `2-1.detail-common.md` の `5.2.2 blogs` 準拠
- 除外項目: `id`, `summary`, `updated_at`
- デフォルト値:
  - `status`: `private`
- リクエスト時の変換項目:
  - `category`: 文字列で受け取る
- 補足:
  - `summary` は本文から自動作成する
- レスポンス: `2-1.detail-common.md` の `5.2.2 blogs` 準拠
- レスポンス時の変換項目:
  - `category`: 文字列で返却する

### 3.6 記事更新API
- 対象API: `PUT /admin/api/blogs/{blog_id}`
- リクエスト: `2-1.detail-common.md` の `5.2.2 blogs` 準拠
- 除外項目: `id`, `summary`, `updated_at`
- リクエスト時の変換項目:
  - `category`: 文字列で受け取る
- 補足:
  - `summary` は本文から自動作成する
- レスポンス: `2-1.detail-common.md` の `5.2.2 blogs` 準拠
- レスポンス時の変換項目:
  - `category`: 文字列で返却する

### 3.7 記事削除API
- 対象API: `DELETE /admin/api/blogs/{blog_id}`
- リクエスト: なし
- レスポンス:
  - `id`: 削除対象の記事ID
  - `result`: `deleted`

### 3.8 画像アップロードAPI
- 対象API: `POST /admin/api/blogs/{blog_id}/images`
- リクエスト形式: `multipart/form-data`
- パスパラメータ:
  - `blog_id`: 紐づけ先の記事ID
- form-data:
  - `file`: 画像ファイル本体
  - `alt_text`: 代替テキスト
- 画像処理:
  - アップロード時に画像をリサイズする
  - 最大サイズは `1920x1080` とする
  - 保存先パスは `<DATA_DIR>/images/<blog_id>/<image_id>.png` とする
- レスポンス:
  - `result`: `success`
  - `url`: 管理画面専用の画像参照用URL `/admin/images/<blog_id>/<image_id>.png`
  - `alt_text`: 代替テキスト

### 3.9 画像削除API
- 対象API: `DELETE /admin/api/blogs/{blog_id}/images/{id}`
- リクエスト: なし
- パスパラメータ:
  - `blog_id`: 紐づけ先の記事ID
  - `id`: 画像ID
- レスポンス:
  - `id`: 削除対象の画像ID
  - `blog_id`: 紐づけ先の記事ID
  - `result`: `deleted`
  - `url` は返却しない

### 3.10 静的HTML出力API
- 対象API: `POST /admin/api/publish`
- リクエスト:
  - `target`: `all` / `index` / `blogs` / `blog` / `about`
  - `blog_id`: `target=blog` の場合は必須、`target=blogs` の場合は任意で指定する
- 必須条件:
  - `target=blog` の場合は `blog_id` を必須とする
- 出力対象ルール:
  - `all`: `index` + `blogs` + `blog詳細` + `about` + `error`
  - `index`: トップページ
  - `blogs`: 記事一覧ページ、記事詳細ページ、カテゴリ別一覧
  - `blog`: 指定した記事詳細ページ
  - `about`: プロフィールページ
- 再生成ルール:
  - `target=blogs` で `blog_id` 未指定時は、記事一覧ページの全ページ、記事詳細ページの全件、カテゴリ別一覧の全カテゴリを再生成する
  - `target=blogs` で `blog_id` 指定時は、記事一覧ページの全ページ、指定した `blog_id` の記事詳細ページ、該当カテゴリのカテゴリ別一覧のみ再生成する
  - 公開用画像は、必要なファイルを `STATIC_EXPORT_DIR/assets/images/` 配下へコピーする
- ログ出力:
  - 生成対象
  - 実行結果
  - 生成ファイル情報
  - 出力先ディレクトリ
  - 上記は専用ログに出力する
- レスポンス:
  - `result`: `success`

## 4. 管理ページレイアウト設計
### 4.1 管理ページレイアウト
- 管理トップページ
- トップ更新ページ
- 記事更新ページ
- 記事本文は Markdown で記載可能とする
- サイト説明は Markdown で記載可能とする
- Markdown エディタは `mmemo` で作成したコンポーネントを流用する
- トップ更新ページと記事更新ページは、共通の編集モード構成を持つ
- 記事更新ページでは、記事公開操作および静的HTML出力操作を行えるようにする

### 4.2 管理ページ コンポーネント分割
- 公開ページは対象外とし、管理ページのみを対象とする
- 共通コンポーネント化する対象は以下とする
  - ヘッダ: `AdminHeader`
  - フッタ: `AdminFooter`
  - Markdown エディタ: `MarkdownEditor`
  - 管理メニュー / タブメニュー: `AdminNavigation`
  - 一覧テーブル: `BlogListTable`
  - ページネーション: `Pagination`
  - フォームアクションボタン: `FormActionButtons`
  - 確認ダイアログ: `ConfirmDialog`
  - 通知メッセージ / エラー表示: `FlashMessage`
  - 画像アップロード部品: `ImageUploader`
- 各ページ固有の項目配置や画面構成は、共通コンポーネントを組み合わせて設計する

### 4.3 管理ページ別 コンポーネント構成
#### 4.3.1 管理トップページ
- 使用する共通コンポーネント
  - ヘッダ: `AdminHeader`
  - フッタ: `AdminFooter`
  - 管理メニュー / タブメニュー: `AdminNavigation`
  - 一覧テーブル: `BlogListTable`
  - ページネーション: `Pagination`
  - 通知メッセージ / エラー表示: `FlashMessage`
- ページ固有コンポーネント
  - 管理トップ概要パネル: `AdminDashboardPanel`
  - 記事一覧セクション: `AdminBlogListSection`
  - 記事更新ページ遷移リンク: `BlogEditLink`
  - トップ更新ページ遷移リンク: `SiteSettingsLink`

#### 4.3.2 トップ更新ページ
- 使用する共通コンポーネント
  - ヘッダ: `AdminHeader`
  - フッタ: `AdminFooter`
  - 管理メニュー / タブメニュー: `AdminNavigation`
  - Markdown エディタ: `MarkdownEditor`
  - フォームアクションボタン: `FormActionButtons`
  - 通知メッセージ / エラー表示: `FlashMessage`
- ページ固有コンポーネント
  - サイト設定フォーム: `SiteSettingsForm`
  - タブ一覧編集セクション: `TabListSection`
  - コピーライト編集セクション: `CopyrightSection`

#### 4.3.3 記事更新ページ
- 使用する共通コンポーネント
  - ヘッダ: `AdminHeader`
  - フッタ: `AdminFooter`
  - 管理メニュー / タブメニュー: `AdminNavigation`
  - Markdown エディタ: `MarkdownEditor`
  - フォームアクションボタン: `FormActionButtons`
  - 確認ダイアログ: `ConfirmDialog`
  - 通知メッセージ / エラー表示: `FlashMessage`
  - 画像アップロード部品: `ImageUploader`
- ページ固有コンポーネント
  - 記事編集フォーム: `BlogEditForm`
  - カテゴリ入力セクション: `CategoryField`
  - 公開状態選択セクション: `StatusField`
  - 公開日入力セクション: `PublishedAtField`
  - 画像一覧セクション: `ImageListSection`
  - プロフィール編集切り替えセクション: `ProfileEditSwitch`

#### 4.3.4 補足
- プロフィールページの更新は記事更新ページの構成を流用する
- `title = about` のレコードを編集する場合は、プロフィール編集モードとして扱う
- 一覧テーブルとページネーションは、管理トップページの記事一覧で使用する
- Markdown エディタは記事本文とサイト説明で共通利用する
