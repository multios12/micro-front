# 公開ページ詳細設計書

## 1. 概要
本書は、`1.site-design.md` および `2-1.detail-common.md` をもとに、公開ページの詳細設計を整理するための文書とする。

## 2. 対象範囲
- 公開ページレイアウト
- 公開URLに関する詳細
- 静的HTML出力後の公開コンテンツ設計

## 3. 公開ページ方針
- 公開ページは静的HTMLとして出力し、HTMLとして公開する
- 公開側では管理APIを使用しない
- 公開ページはAPIレスポンスではなく、生成済みファイルを返却する
- 公開側の詳細レイアウトは、必要に応じて別途設計する

## 4. 公開ページレイアウト
### 4.1 共通レイアウト
- ヘッダ
  - サイトタイトル
  - サイトサブタイトル
- タブナビゲーション
- コンテンツ領域
- フッタ
  - フッタ情報
  - コピーライト

### 4.2 対象ページ
- トップページ
- 記事一覧ページ
- 記事詳細ページ
- プロフィールページ
- errorページ

### 4.3 補足
- レイアウトの詳細デザインは別途設計する
- 公開ページでは、`1.site-design.md` の公開ページ要件を優先する

## 5. 公開コンテンツ出力
### 5.1 静的HTML出力対象
- `index`: トップページ
- `blogs`: 記事一覧ページ
- `blog`: 指定した記事詳細ページ
- `about`: プロフィールページ
- `all`: `index` + `blogs` + `blog詳細` + `about` + `error`

### 5.2 出力ルール
- 公開ページは静的HTMLとして出力する
- 出力先ディレクトリは `STATIC_EXPORT_DIR` とし、デフォルト値は `./data/publish` とする
- 画像は公開HTMLとは別にファイル参照で表示する
- 公開用画像は、`publish` 実行時に `<DATA_DIR>/images/<blog_id>/<image_id>.png` から `STATIC_EXPORT_DIR/assets/images/<blog_id>/<image_id>.png` へコピーする
- 出力結果の詳細は専用ログへ記録する

### 5.3 静的HTMLディレクトリ構成
- 出力先ルートは `STATIC_EXPORT_DIR` とし、デフォルト値は `./data/publish` とする
- URL と HTML ファイルの対応は、次の構成を基本とする
  - `/` : `index.html`
  - `/blogs` : `blogs/index.html`
  - `/blogs/page{n}` : `blogs/page{n}.html`
  - `/blogs/category/{category}` : `blogs/category/{category}/index.html`
  - `/blogs/category/{category}/page{n}` : `blogs/category/{category}/page{n}.html`
  - `/blogs/{blog_id}` : `blogs/{blog_id}.html`
  - `/about` : `about/index.html`
  - `/error` : `error.html`
- ページングは、1ページ目を `blogs/index.html`、2ページ目以降を `blogs/page{n}.html` とする
- カテゴリ別一覧は、カテゴリごとに静的HTMLを事前生成する
- カテゴリ別ページングは、1ページ目を `blogs/category/{category}/index.html`、2ページ目以降を `blogs/category/{category}/page{n}.html` とする
- 記事一覧ページの1ページあたり表示件数は 20 件とし、環境変数で変更可能とする
- 画像などの静的資産は `assets/` 配下に配置する
  - 画像: `assets/images/<blog_id>/<image_id>.png`

### 5.4 再生成規約
- `target=index` はトップページを再生成する
- `target=about` はプロフィールページを再生成する
- `target=blog` は指定した `blog_id` の記事詳細ページを再生成する
- `target=blogs` は記事一覧ページ、記事詳細ページ、カテゴリ別一覧を再生成する
- `target=blogs` で `blog_id` が指定されていない場合:
  - トップページも再生成する
  - 記事一覧ページは全ページを再生成する
  - 記事詳細ページは全件を再生成する
  - カテゴリ別一覧は全カテゴリを再生成する
- `target=blogs` で `blog_id` が指定されている場合:
  - トップページも再生成する
  - 記事一覧ページは全ページを再生成する
  - 記事詳細ページは指定した `blog_id` のみ再生成する
  - カテゴリ別一覧は指定した `blog_id` の記事が属するカテゴリのみ再生成する
- `target=all` は `index`、`blogs`、`about`、`error` をまとめて再生成する
- 記事削除やカテゴリ変更により不要になった公開HTMLおよび公開用画像は、再生成時に物理削除する

## 6. 公開URL補足
- トップページ: `/`
- 記事一覧ページ: `/blogs`
- 記事詳細ページ: `/blogs/{blog_id}`
- プロフィールページ: `/about`
- カテゴリ別記事一覧ページ: `/blogs/category/{category}`
- キーワード検索クエリ `q` はひとまず実装対象外とする
- `category` は1件のみ指定可能とし、完全一致で絞り込む
- トップページの新着記事は `public` の記事のみを対象とし、`published_at` の降順で 20 件表示する

### 6.1 補足
- 公開ページは Go 言語の App サーバまたは静的配信基盤から返却する。
- 配信対象は生成済み HTML と静的アセットのみとし、公開側では管理 API を公開しない。
