# 基本設計書

## 1. 概要
シンプルな個人ブログサイトを作成する。  
管理ページで記事やサイト情報を更新した後、公開ページ用の静的HTMLを出力し、HTMLとして公開する構成とする。

## 2. 対象ページ
- トップページ
- 記事一覧ページ
- 記事詳細ページ
- プロフィールページ
- errorページ
- 管理トップページ
- トップ更新ページ
- 記事更新ページ

## 3. 機能概要
- 記事一覧表示
- 記事詳細表示
- ページネーション
- カテゴリによる絞り込み
- 公開ページ用静的HTMLの出力

## 4. 公開ページ要件
### 4.1 共通レイアウト
- 画面上部のヘッダにサイトタイトルを表示する
- ヘッダにタブを表示し、ページを切り替え可能にする
- タブの下に各ページのコンテンツを表示する
- 画面下部のフッタにコピーライトを表示する
- サイトタイトル、タブ、切り替え先URL、コピーライトはデータ管理し、差し替え可能とする

### 4.2 トップページ
- サイトタイトルを表示する
- 新着記事を表示する
- 記事一覧ページへのリンクを配置する
- プロフィールページへのリンクを配置する

補足:
- 新着記事は `public` の記事のみを対象とする
- 表示順は `published_at` の降順とする
- 表示件数は 20 件とし、環境変数で変更可能とする

### 4.3 記事一覧ページ
- 記事タイトルを一覧表示する
- 公開日を表示する
- カテゴリを表示する
- カテゴリをクリックすることで、同じカテゴリの記事に絞り込みできる
- 各記事詳細ページへ遷移できる
- ページネーションを表示する

補足:
- 一覧表示対象は `public` の記事のみとする
- 表示順は `published_at` の降順とする
- 1ページあたりの表示件数は 20 件とし、環境変数で変更可能とする

### 4.4 記事詳細ページ
- 記事タイトルを表示する
- 公開日を表示する
- カテゴリを表示する
- カテゴリをクリックすることで、同じカテゴリの記事に絞り込みできる
- 本文を表示する
- 本文内に画像を埋め込みできる

### 4.5 プロフィールページ
- 自己紹介文を表示する
- ブログの説明を表示する

### 4.6 errorページ
- ページが見つからないことを表示する
- トップページへのリンクを設置する

## 5. 管理ページ要件
### 5.1 管理トップページ
- トップ更新ページへ切り替え可能にする
- 記事の一覧を表示する
- 各記事から記事更新ページへ切り替え可能にする

### 5.2 トップ更新ページ
- サイト名を変更可能にする
- サイト説明を変更可能にする
- コピーライトを変更可能にする

### 5.3 記事更新ページ
- 記事の更新を可能にする
- 画像のアップロードを可能にする
- プロフィールも記事として更新可能にする
- 記事公開操作を可能にする
- 更新後に公開ページ用の静的HTMLを出力可能にする

## 6. URL要件
### 6.1 公開ページURL要件
- `/` : トップページ
- `/blogs` : 記事一覧ページ
- `/blogs/{blog_id}` : 記事詳細ページ
- `/about` : プロフィールページ

補足:
- 公開ページは静的HTMLとして出力したファイルを公開する
- プロフィールページ `/about` は通常記事URL仕様の対象外とする

### 6.2 一覧・ページング仕様案
- 記事一覧ページのURLは `/blogs` とする
- 記事一覧ページはページネーションに対応する
- カテゴリ別記事一覧ページのURLは `/blogs/category/{category}` とする

補足:
- `q` はひとまず実装対象外とする
- `category` は1件のみ指定可能とし、完全一致で絞り込む
- 通常の記事一覧は静的HTMLとして事前生成する
- カテゴリ別一覧は静的HTMLとして事前生成する
- 1ページあたりの表示件数は 20 件とし、環境変数で変更可能とする

### 6.3 管理ページURL要件
- `/admin` : 管理トップページ
- `/admin/top` : トップ更新ページ
- `/admin/blogs/{blog_id}` : 記事更新ページ

## 7. データ設計
### 7.1 共通レイアウト設定
- `site_title: string` / サイトタイトル / 100文字
- `site_subtitle: string` / サイトサブタイトル / 100文字
- `site_description: string` / サイト説明 / 1000文字
- `tabs: object[]` / タブ一覧 / 最大10件
  - `tab_label: string` / タブラベル / 20文字
  - `tab_url: string` / タブ切り替え先URL / 100文字
- `foot_information: string` / フッタ情報 / 100文字
- `copyright: string` / コピーライト / 100文字

補足:
- `site_title` はヘッダに表示するタイトルとする
- `site_subtitle` はヘッダに表示するサブタイトルとする
- `tabs` はヘッダに表示するナビゲーション項目とする
- `tab_url` は `/`、`/blogs`、`/about` などの内部URLを想定する
- `foot_information` はフッタ左側に表示する情報とする

### 7.2 記事
論理名: `blogs`  
単体レコード名: `blog`

- `id: number` / 記事ID / 10桁
- `title: string` / 記事タイトル / 100文字
- `content: string` / 本文 / 20,000文字
- `summary: string` / 概要 / 200文字
- `category: string` / カテゴリ / 20文字
- `status: string` / 公開状態 / 7文字
- `published_at: datetime` / 公開日 / 19文字
- `updated_at: datetime` / 更新日 / 19文字

補足:
- `status` は以下のコードを使用する
  - `public`: 公開
  - `private`: 未公開
- `published_at` は変更可能
- `updated_at` は自動入力
- 記事本文内では画像を埋め込み可能とする
- 画像データは記事データと別に管理する

### 7.3 プロフィールページ管理
- プロフィールページは、記事データの中の特別な1レコードとして管理する
- 使用するデータ項目は `7.2 記事` と同一とする
- プロフィールページ用レコードは1件のみ保持する
- `title` は `about` とする
- URL は `/about` 固定とする
- `/blogs` の一覧には表示しない
- プロフィールページでは `category` は使用しない

### 7.4 画像
論理名: `images`  
単体レコード名: `image`

- `id: number` / 画像ID / 10桁
- `blog_id: number` / 記事ID / 10桁
- `alt_text: string` / 代替テキスト / 200文字
- `created_at: datetime` / 作成日 / 19文字
- `updated_at: datetime` / 更新日 / 19文字

補足:
- 画像データは記事データと別管理とする
- `blog_id` により、どの記事に紐づく画像かを管理する

## 8. デザイン方針
- シンプルで読みやすいデザイン
- 管理しやすい構成
- スマートフォンでも見やすいレイアウトにする

## 9. 開発ステップ
1. 公開ページ：HTMLモック作成・デザイン調整
2. 管理ページ：HTMLモック作成・デザイン調整
3. 開発環境すなわち、dev container整備
4. DB実装
5. 管理ページ：トップページ作成
6. 管理ページ：トップ更新ページ作成
7. 管理ページ：記事更新ページ作成
8. 管理ページ：画像アップロード機能実装
9. 公開ページ：静的HTML出力機能実装。静的HTMLのテンプレートにはHTMLモックを使用
