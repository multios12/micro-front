# ブログタイトル画像ジェネレータ詳細設計書

## 1. 概要
本書は、ブログ記事のタイトル画像を生成するためのコンポーネント設計を整理するための文書とする。  
タイトル画像は、背景画像と記事タイトルを組み合わせた SVG として生成し、記事ごとに選択可能なテンプレートをもとに出力する。

## 2. 対象範囲
- タイトル画像ジェネレータコンポーネント
- タイトル画像テンプレート
- SVG生成仕様
- 管理ページでのテンプレート選択・プレビュー
- 静的HTML出力時に利用するタイトル画像ファイル

補足:
- 本書では、タイトル画像の生成仕様を中心に定義する
- 記事本文内の画像アップロード機能は対象外とする
- AI画像生成や外部画像検索による背景画像生成は当初実装対象外とする

## 3. 基本方針
- タイトル画像は SVG 形式で生成する
- 画像サイズは 横1200px × 縦675px とする
- アスペクト比は 16:9 とする
- 背景画像または背景パターンと、記事タイトルのテキストで構成する
- 複数種類のテンプレートを用意し、記事ごとに選択可能とする
- 当初は固定テンプレートを使用し、ユーザーによる細かなデザイン編集は実装しない

補足:
- SVG とすることで、テキスト・図形・背景パターンをコードとして管理しやすくする
- 公開ページで扱いやすいよう、生成結果は静的ファイルとして出力する
- OGP画像としての利用も想定できるサイズにする

## 4. 画面要件
### 4.1 管理ページ 記事更新ページ
- 記事更新ページにタイトル画像テンプレート選択欄を表示する
- テンプレート選択欄では、利用可能なテンプレートを一覧表示する
- 選択中テンプレートをもとにタイトル画像プレビューを表示する
- 記事タイトルを変更した場合、プレビューへ反映する
- 新規記事作成時のテンプレート選択欄のデフォルト値は `diary` とする

補足:
- プレビューは管理ページ上で即時確認できるようにする
- タイトル画像の生成元テキストは `blogs.title` を使用する
- システムの設計上、タイトルは入力必須のため、未入力は考慮しない
- 記事保存時に、選択したテンプレートIDを blogs.title_image_template として保存する
- 既存記事で `blogs.title_image_template` が未設定の場合は、管理ページ上では `diary` を選択状態として扱う
- プロフィールページ用の記事はタイトル画像生成の対象外とする

### 4.2 公開ページ
- 記事詳細ページで、生成済みタイトル画像を表示可能とする
- 記事一覧ページでは、生成済みタイトル画像をサムネイルとして表示する
- トップページでは、新着記事のタイトル画像をサムネイルとして表示する
- `meta property="og:image"` への設定は将来拡張とする

## 5. テンプレート仕様
### 5.1 テンプレート一覧
当初は下記4種類のテンプレートを実装する。

| テンプレートID | 表示名 | コンセプト | 想定カテゴリ |
| --- | --- | --- | --- |
| `tech` | Tech | 無機質・発光。ターミナルをイメージしたもの | 技術記事 |
| `book` | Book | 紙・古書 | 読書、書評、知識整理 |
| `diary` | Diary | 淡い抽象 | 日記、雑記 |
| `travel` | Travel | 地図・等高線 | 旅行、場所、体験記 |

### 5.2 共通レイアウト
- SVG の `viewBox` は `0 0 1200 675` とする
- 背景はテンプレートごとに図形、グラデーション、パターンで構成する
- 記事タイトルはSVG内のテキストとして描画する
- タイトル画像内にはサイトタイトルやカテゴリ名は表示しない
- タイトルは読みやすさを優先し、背景とのコントラストを確保する
- タイトル表示領域は、上下左右に十分な余白を設ける

補足:
- 長いタイトルは複数行に折り返す
- 折り返し後も表示領域に収まらない場合は、フォントサイズを段階的に小さくする
- SVGの外部フォント依存は避け、標準的なフォントファミリー指定に留める
- テンプレートごとの色や余白は現時点では設定可能にしない
- ユーザーによる背景画像アップロードは現時点では想定しない

### 5.3 `tech` テンプレート
- 黒または濃いグレーを基調とする
- ターミナル画面、コード、走査線、グリッドを連想する背景にする
- アクセント色としてシアン、グリーン、ブルー系の発光表現を使用する
- タイトルは明るい色で表示する
- 直線、細い罫線、矩形を中心に構成する

### 5.4 `book` テンプレート
- 紙、古書、余白、活字を連想する背景にする
- 生成色はアイボリー、ブラウン、インク色を中心にする
- タイトルは紙面上の見出しのように配置する
- 装飾は控えめにし、紙の質感や罫線を使う
- 過度な古びた表現で可読性を落とさない

### 5.5 `diary` テンプレート
- 淡い抽象背景を使用する
- 明るく柔らかい色を中心にする
- 円形、ぼかし風の図形、薄い線を使い、静かな印象にする
- タイトルは落ち着いた濃色で表示する
- 個人ブログの雑記に使いやすい汎用的な見た目にする

### 5.6 `travel` テンプレート
- 地図、等高線、経路、座標を連想する背景にする
- ベース色は生成り、淡い緑、青、グレーなどを使用する
- 等高線風の曲線、地点マーカー、破線を配置する
- タイトルは地図ラベルのように読みやすく配置する
- 写真を使わず、抽象化された地図表現を中心にする
- 地図データを利用する場合は OpenStreetMap を候補とする
- 当初は固定の抽象地図表現とし、場所指定・外部地図データ取得は将来拡張とする

## 6. データ設計
### 6.1 blogs 追加項目案
`blogs` テーブルに下記項目を追加する。

| カラム名              | 型   | 必須 | 説明                      |
| -------------------- | ---- | ---- | ------------------------ |
| title_image_template | TEXT | 必須 | タイトル画像テンプレートID |

補足:
- `title_image_template` は `tech` / `book` / `diary` / `travel` のいずれかとする
- デフォルト値は `diary` とする
- 将来テンプレート追加時に、値の追加を行う

### 6.2 DBマイグレーション
既存DBに対して、`blogs` テーブルへ `title_image_template` を追加する。

ALTER TABLE案:
```sql
ALTER TABLE blogs
ADD COLUMN title_image_template TEXT NOT NULL DEFAULT 'diary';
```

補足:
- 既存記事には `diary` を設定する
- アプリ起動時のスキーマ初期化では、既存の `site_url` 追加と同様に重複追加を許容する移行処理とする
- 新規作成される `blogs` テーブル定義にも `title_image_template TEXT NOT NULL DEFAULT 'diary'` を含める

### 6.3 API項目
記事取得・保存APIに下記項目を追加する。

- `title_image_template: string`

補足:
- 既存記事で値が未設定の場合は `diary` として扱う
- 不正なテンプレートIDが指定された場合はバリデーションエラーとする

## 7. SVG生成仕様
### 7.1 入力値
- `title`: 記事タイトル
- `template`: テンプレートID

補足:
- `siteTitle` やカテゴリ名は入力値として扱わない
- プロフィールページ用の記事はSVG生成対象外とする

### 7.2 出力
- SVG文字列を生成する
- 静的HTML出力時は、SVGファイルとして保存する
- ファイル名は `{blog_id}.svg` とする
- 保存先は公開用アセットディレクトリ配下とする

出力パス案:
- `assets/title-images/{blog_id}.svg`

補足:
- 公開ページの画像参照は、各HTMLファイルから `assets/title-images/{blog_id}.svg` へ相対パスで参照できるようにする
- 記事詳細ページ、記事一覧ページ、トップページはいずれも同じSVGファイルを参照する
- 将来 OGP 用に PNG 変換が必要になった場合は、SVG生成後に別処理で変換する
- 初期実装では `og:image` 用の PNG 変換は行わない

### 7.3 テキスト描画
- タイトルは最大3行を目安に折り返す
- 折り返しはバイト数ではなく rune 単位で行う
- 日本語などの全角文字は幅2、英数字などの半角文字は幅1として概算し、表示幅に基づいて折り返す
- 1行あたりの表示幅はテンプレートごとのタイトル表示領域に応じて調整する
- 初期値として、最大3行、基準フォントサイズ56px、最小フォントサイズ38pxを目安とする
- 最大行数に収まらない場合は、フォントサイズを段階的に小さくする
- 最小フォントサイズでも収まらない場合は、最終行の末尾を `...` で省略する
- 改行位置は日本語・英数字混在でも破綻しにくいようにする
- SVG内に直接ユーザー入力文字列を埋め込むため、HTMLエスケープを行う

補足:
- `<`, `>`, `&`, `"`, `'` はエスケープ対象とする
- サンプルSVG確認時に、折り返し位置、行間、フォントサイズをテンプレートごとに調整する

## 8. コンポーネント設計
### 8.1 管理ページコンポーネント
コンポーネント案:
- `TitleImageTemplateSelector.svelte`
- `TitleImagePreview.svelte`

`TitleImageTemplateSelector.svelte`
- テンプレート一覧を表示する
- 選択中テンプレートを変更する
- 親コンポーネントへ選択値を通知する

`TitleImagePreview.svelte`
- 記事タイトルとテンプレートIDを受け取る
- SVGプレビューを表示する
- 管理ページ上では横幅に合わせて縮小表示する

### 8.2 SVG生成ロジック
SVG生成ロジックの本体は Go 側に実装する。

Go側ロジック案:
- `GenerateTitleImageSVG(input): (string, error)`
- `ListTitleImageTemplates(): []TitleImageTemplate`

型定義案:
```go
type TitleImageTemplateID string

const (
	TitleImageTemplateTech   TitleImageTemplateID = "tech"
	TitleImageTemplateBook   TitleImageTemplateID = "book"
	TitleImageTemplateDiary  TitleImageTemplateID = "diary"
	TitleImageTemplateTravel TitleImageTemplateID = "travel"
)

type TitleImageGenerateInput struct {
	Title    string
	Template TitleImageTemplateID
}

type TitleImageTemplate struct {
	ID          TitleImageTemplateID
	Label       string
	Description string
}
```

補足:
- 管理ページのプレビューと静的HTML出力処理は、同じGo側SVG生成ロジックを使用する
- フロントエンド側ではSVG生成ロジックを持たず、テンプレート選択UIとプレビュー表示を担当する
- これにより、管理ページのプレビューと公開ページに出力されるSVGの差異を抑える
- サムネイルURLはDBやAPIレスポンスの永続データとして保持せず、公開HTML生成時に記事IDから自動生成する
- 公開HTML生成時は `relURL(pageFile, "assets/title-images/{blog_id}.svg")` 相当の処理で、各ページからの相対URLを生成する
- サムネイルURL生成は `titleImageURL(pageFile, blogID)` のような共通helperとして実装する
- 記事詳細ページ、記事一覧ページ、トップページはいずれも同じhelperでタイトル画像URLを生成する

### 8.3 SVGテンプレート管理
SVGテンプレートは Go 側のタイトル画像生成パッケージ内で管理する。

管理方針:
- テンプレートID、表示名、説明は一覧定義として管理する
- 各テンプレートのSVG描画処理はテンプレートIDごとの関数として実装する
- 共通のSVGヘッダ、サイズ、テキスト折り返し、HTMLエスケープ処理は共通関数化する
- Goの `html/template` を主用途には使用しない

補足:
- SVGは固定HTMLページよりも図形・座標・複数行テキストの制御が多いため、当初はGoの関数で生成する
- 記事タイトルなどのユーザー入力文字列は、SVGへ埋め込む直前に必ずエスケープする
- 将来、SVGの静的部分が大きくなった場合は、`.svg.tmpl` ファイルを `go:embed` で読み込み、Go側で描画する方式を検討する

### 8.4 テンプレート一覧API
管理ページのテンプレート選択欄は、API経由でテンプレート一覧を取得する。

API案:
- `GET /admin/api/title-image/templates`

レスポンス:
```json
{
  "items": [
    {
      "id": "diary",
      "label": "Diary",
      "description": "淡い抽象"
    }
  ]
}
```

補足:
- テンプレート一覧はGo側の `ListTitleImageTemplates()` を正とする
- フロントエンド側ではテンプレートID、表示名、説明を固定定義しない
- 新規記事作成時や未設定記事のデフォルト選択は `diary` とする

### 8.5 プレビューAPI
管理ページのタイトル画像プレビューは、API経由で生成したSVGを表示する。

API案:
- `POST /admin/api/title-image/preview`

リクエスト:
```json
{
  "title": "記事タイトル",
  "template": "diary"
}
```

レスポンス:
```json
{
  "svg": "<svg ...>...</svg>"
}
```

補足:
- レスポンス形式は管理ページで扱いやすいよう JSON とする
- API内部では静的HTML出力時と同じSVG生成ロジックを使用する
- 不正なテンプレートIDが指定された場合はバリデーションエラーを返却する
- フロントエンドでは返却されたSVG文字列をBlob URLまたはdata URLに変換し、`<img>` で表示する
- SVG文字列を管理ページHTMLへ直接インライン埋め込みしない

## 9. バリデーション
- 保存APIでは `title_image_template` が未指定の場合、`diary` に補完する
- 補完後の `title_image_template` は許可されたテンプレートIDのみ保存可能とする
- 不正なテンプレートIDが指定された場合はバリデーションエラーとする
- SVG生成時のタイトル文字列は必ずエスケープする

エラーコード追加案:
- `INVALID_TITLE_IMAGE_TEMPLATE`: タイトル画像テンプレートID不正

## 10. 静的HTML出力
- 記事詳細ページ出力時に、タイトル画像SVGを同時に出力する
- 記事一覧ページ出力時に、各記事のタイトル画像SVGをサムネイルとして参照する
- トップページ出力時に、新着記事のタイトル画像SVGをサムネイルとして参照する
- 記事タイトルまたはテンプレートが変更された場合は、次回 publish 時に再生成する
- 出力先に既存SVGがある場合は上書きする

補足:
- SVGは記事データから再生成可能な派生ファイルとして扱う
- DBにはSVG文字列本体を保存しない
- タイトル画像SVGは `STATIC_EXPORT_DIR/assets/title-images/{blog_id}.svg` に出力する
- 公開ページのHTML側には `<img>` タグで参照する
- SVGはHTMLへ直接埋め込まず、生成済みSVGファイルを参照する
- サムネイルURLはテンプレート表示用データとして固定保持せず、記事IDとページ位置から相対URLを自動生成する
- プロフィールページ用の記事はタイトル画像SVGを出力しない
- `og:image` 用の PNG ファイルは初期実装では出力しない

## 11. 開発ステップ
1. SVG生成ロジックを実装する
2. サンプルSVG自動生成用のアプリ引数を追加する
3. 各テンプレートのサンプルSVGを生成し、`docs/mocks/blog-header` に格納する
4. サンプルSVGを確認し、テンプレートの見た目を調整する
5. `blogs` テーブルに `title_image_template` を追加する
6. 記事取得・保存APIに `title_image_template` を追加する
7. 管理ページの記事更新フォームにテンプレート選択欄を追加する
8. 管理ページにタイトル画像プレビューを追加する
9. 静的HTML出力時にタイトル画像SVGを出力する
10. 記事詳細ページでタイトル画像を表示する
11. 記事一覧ページとトップページでタイトル画像をサムネイルとして表示する

補足:
- サンプルSVG自動生成用のアプリ引数は `sample` とする
- 実行例: `go run ./cmd/micro-front sample -output-dir docs/mocks/blog-header -title "ブログタイトル画像ジェネレータ"`
- サンプルSVGは `tech.svg` / `book.svg` / `diary.svg` / `travel.svg` として出力する
- サンプル確認では、タイトルの可読性、余白、色のコントラスト、サムネイル縮小時の見え方を確認する

## 12. テスト観点
- SVG生成ロジックで、4種類のテンプレートすべてのSVG文字列を生成できること
- SVG生成時に、記事タイトル内の `<`, `>`, `&`, `"`, `'` がエスケープされること
- 不正なテンプレートIDを指定した場合、バリデーションエラーになること
- `blogs.title_image_template` 追加後、既存記事の値が `diary` として扱われること
- 記事取得・保存APIで `title_image_template` を取得・更新できること
- 管理ページの新規記事作成時、テンプレート選択欄のデフォルトが `diary` になること
- プレビューAPIで、保存前の記事タイトルとテンプレートIDからSVGを生成できること
- `sample` 実行時に、`docs/mocks/blog-header` へ `tech.svg` / `book.svg` / `diary.svg` / `travel.svg` が出力されること
- publish 実行時に、通常記事のタイトル画像SVGが `STATIC_EXPORT_DIR/assets/title-images/{blog_id}.svg` に出力されること
- プロフィールページ用の記事では、タイトル画像SVGが出力されないこと
- 記事詳細ページ、記事一覧ページ、トップページで、タイトル画像SVGが `<img>` タグで参照されること
- 記事詳細ページ、記事一覧ページ、トップページから、タイトル画像SVGへの相対URLが正しいこと
- 公開HTMLにSVG文字列が直接インライン埋め込みされないこと
- `og:image` 用のPNGファイルが初期実装では出力されないこと

## 13. 決定事項
- タイトル画像は記事一覧ページとトップページのサムネイルとして利用する
- タイトル画像内にはサイトタイトルやカテゴリ名を表示しない
- タイトル画像生成の入力値にサイトタイトルやカテゴリ名は含めない
- サムネイルURLは記事データとして保持せず、記事IDから自動生成する
- 公開HTMLではSVGを直接埋め込まず、生成済みSVGファイルを `<img>` で参照する
- プロフィールページ用の記事はタイトル画像生成の対象外とする
- テンプレートごとの色や余白は現時点では設定可能にしない
- ユーザーによる背景画像アップロードは現時点では想定しない
- `og:image` 用の PNG 変換は初期実装では行わない

## 14. 将来拡張
- `og:image` 用にタイトル画像を PNG 変換する
- `travel` テンプレートで場所指定や外部地図データ取得に対応する
