# カテゴリ管理詳細設計書

## 1. 概要
本書は、管理画面にカテゴリ管理機能を追加するための詳細設計を整理するための文書とする。  
カテゴリは専用のカテゴリテーブルで管理し、トップページに表示するカテゴリ一覧はカテゴリテーブルの内容をもとに出力する。

## 2. 対象範囲
- カテゴリ管理用DBテーブル
- カテゴリ管理API
- 管理画面のカテゴリ管理ページ
- 記事編集画面でのカテゴリ選択
- トップページのカテゴリ一覧出力
- 静的HTML出力時のカテゴリ表示ルール

補足:
- 記事詳細ページ、記事一覧ページ、カテゴリ別記事一覧ページの基本構成は既存仕様を維持する
- 記事に保存するカテゴリ値は、既存の `blogs.category` を継続利用する
- 本書では複数カテゴリ付与は扱わず、1記事につき1カテゴリの仕様を維持する

## 3. 基本方針
- カテゴリは `categories` テーブルで管理する
- カテゴリは親カテゴリ名、子カテゴリ名、並び順を持つ
- 親カテゴリ名のみのカテゴリも作成可能とする
- 子カテゴリ名がある場合は、親カテゴリ配下の子カテゴリとして扱う
- トップページに表示されるカテゴリは、公開済み記事からの自動抽出ではなく、`categories` テーブルの内容をそのまま出力する
- 表示順は `sort_order` の昇順とする
- 同一の並び順の場合は、親カテゴリ名、子カテゴリ名の昇順で安定化する

補足:
- 既存実装では `blogs.category` からカテゴリ一覧を生成しているが、本機能追加後はトップページカテゴリ一覧の正は `categories` テーブルとする
- `categories` に登録されているカテゴリは、該当記事が0件でもトップページに表示する
- カテゴリ別記事一覧ページへのリンク先は、従来どおりカテゴリ名から生成したURLを使用する

## 4. データ設計
### 4.1 categories テーブル
- テーブル名: `categories`

| カラム名 | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| id | INTEGER | 必須 | 主キー |
| parent_name | TEXT | 必須 | 親カテゴリ名 |
| child_name | TEXT | 任意 | 子カテゴリ名 |
| sort_order | INTEGER | 必須 | 並び順 |
| updated_at | TEXT | 必須 | 更新日時 |

補足:
- `parent_name` はトップページのカテゴリグループ名として使用する
- `child_name` が空の場合、その行は親カテゴリ単体のカテゴリとして扱う
- `child_name` がある場合、その行は親カテゴリ配下の子カテゴリとして扱う
- `sort_order` は小さい値ほど上に表示する
- `updated_at` は更新時に自動更新する

### 4.2 カテゴリ表示名
記事に保存するカテゴリ文字列およびカテゴリ別記事一覧URLに使用するカテゴリ名は、下記ルールで生成する。

| 条件 | 表示名 |
| --- | --- |
| `child_name` が空 | `parent_name` |
| `child_name` がある | `parent_name + " / " + child_name` |

補足:
- `blogs.category` にはカテゴリ表示名を保存する
- 既存記事との互換性を優先し、初期実装では `blogs` から `categories` への外部キーは設定しない
- 将来、カテゴリID参照に移行する場合は別途設計する

### 4.3 制約
- `parent_name` は空文字不可とする
- `child_name` は空文字可とする
- `sort_order` は0以上の整数とする
- `parent_name` と `child_name` の組み合わせは一意とする
- 親カテゴリ単体と子カテゴリは別カテゴリとして扱う

一意制約案:
```sql
CREATE UNIQUE INDEX idx_categories_name
  ON categories (parent_name, child_name);
```

### 4.4 CREATE TABLE SQL案
```sql
CREATE TABLE categories (
  id INTEGER PRIMARY KEY,
  parent_name TEXT NOT NULL,
  child_name TEXT NOT NULL DEFAULT '',
  sort_order INTEGER NOT NULL DEFAULT 0,
  updated_at TEXT NOT NULL
);

CREATE UNIQUE INDEX idx_categories_name
  ON categories (parent_name, child_name);

CREATE INDEX idx_categories_sort_order
  ON categories (sort_order, parent_name, child_name);
```

### 4.5 DBマイグレーション
既存DBに対して、`categories` テーブルを追加する。

補足:
- アプリ起動時のスキーマ初期化で `CREATE TABLE IF NOT EXISTS` を実行する
- 既存の `blogs.category` に保存されているカテゴリから初期カテゴリを生成する移行処理は任意とする
- 初期カテゴリを生成する場合は、重複を除外し、`sort_order` はカテゴリ名昇順で連番を設定する
- `blogs.category` の値は変更しない

## 5. 管理API設計
### 5.1 カテゴリ一覧取得API
- 対象API: `GET /admin/api/categories`
- レスポンス:
  - `items`: カテゴリ配列

`items[]`:

| 項目 | 型 | 説明 |
| --- | --- | --- |
| id | number | カテゴリID |
| parent_name | string | 親カテゴリ名 |
| child_name | string | 子カテゴリ名 |
| sort_order | number | 並び順 |
| display_name | string | 表示用カテゴリ名 |
| updated_at | string | 更新日時 |

補足:
- `items` は `sort_order`, `parent_name`, `child_name` の順で返却する
- `display_name` は API 側で生成して返却する

### 5.2 カテゴリ新規作成API
- 対象API: `POST /admin/api/categories`
- リクエスト:
  - `parent_name`: 親カテゴリ名
  - `child_name`: 子カテゴリ名
  - `sort_order`: 並び順
- レスポンス: 作成後のカテゴリ

### 5.3 カテゴリ更新API
- 対象API: `PUT /admin/api/categories/{category_id}`
- リクエスト:
  - `parent_name`: 親カテゴリ名
  - `child_name`: 子カテゴリ名
  - `sort_order`: 並び順
- レスポンス: 更新後のカテゴリ

補足:
- カテゴリ名を変更しても、既存記事の `blogs.category` は自動更新しない
- 既存記事のカテゴリ変更が必要な場合は、記事編集画面から個別に更新する

### 5.4 カテゴリ削除API
- 対象API: `DELETE /admin/api/categories/{category_id}`
- リクエスト: なし
- レスポンス:
  - `id`: 削除対象のカテゴリID
  - `result`: `deleted`

補足:
- カテゴリを削除しても、既存記事の `blogs.category` は自動更新しない
- 削除対象カテゴリを使用している記事が存在する場合、初期実装では削除可能とする
- 削除後も記事側にカテゴリ文字列が残るため、カテゴリ別記事一覧ページ自体は従来どおり生成可能とする

### 5.5 バリデーション
- `parent_name`
  - 必須
  - 前後の空白を除去して保存する
  - 20文字以内
  - 公開URL生成に使用できない文字は不可
- `child_name`
  - 任意
  - 前後の空白を除去して保存する
  - 20文字以内
  - 公開URL生成に使用できない文字は不可
- `sort_order`
  - 必須
  - 0以上の整数
- `parent_name` + `child_name`
  - 重複不可

エラーコード案:
- `CATEGORY_NAME_CONFLICT`: カテゴリ名重複
- `CATEGORY_NOT_FOUND`: カテゴリなし
- `INVALID_CATEGORY_NAME`: カテゴリ名不正
- `INVALID_CATEGORY_SORT_ORDER`: 並び順不正

## 6. 管理画面設計
### 6.1 カテゴリ管理ページ
管理画面にカテゴリ管理ページを追加する。

画面要素:
- カテゴリ一覧
- 新規カテゴリ追加フォーム
- カテゴリ編集フォーム
- カテゴリ削除操作
- 並び順入力
- 保存結果の通知

一覧表示項目:
- 親カテゴリ名
- 子カテゴリ名
- 表示用カテゴリ名
- 並び順
- 更新日時
- 操作ボタン

補足:
- カテゴリ一覧は `sort_order` の昇順で表示する
- 並び順は数値入力とする
- 新規追加時の `sort_order` は既存最大値 + 10 を初期値とする
- 削除時は確認ダイアログを表示する

### 6.2 記事編集画面
記事編集画面のカテゴリ入力は、カテゴリ管理APIから取得したカテゴリ一覧を選択候補として表示する。

補足:
- 既存記事の `blogs.category` がカテゴリ一覧に存在しない場合でも、現在値として表示する
- 未登録カテゴリの直接入力を許可するかどうかは実装時に選択可能とする
- 初期実装では、既存の自由入力互換を維持するため直接入力を許可する
- カテゴリ選択時は `display_name` を `blogs.category` に保存する

## 7. 公開ページ出力設計
### 7.1 トップページカテゴリ一覧
トップページに表示するカテゴリ一覧は、`categories` テーブルの内容をそのまま出力する。

出力ルール:
- `categories` の全レコードを対象とする
- `sort_order` の昇順で出力する
- `parent_name` ごとにグルーピングする
- `child_name` が空の行は親カテゴリ単体リンクとして表示する
- `child_name` がある行は親カテゴリ配下の子カテゴリリンクとして表示する
- 該当記事が0件のカテゴリも表示する

補足:
- 公開済み記事のカテゴリからトップページカテゴリ一覧を自動生成しない
- 記事件数表示を行う場合でも、カテゴリの表示有無と表示順は `categories` テーブルを正とする
- カテゴリテーブルが空の場合は、既存と同様に「カテゴリはまだありません。」相当の表示とする

### 7.2 カテゴリ別記事一覧ページ
カテゴリ別記事一覧ページは、従来どおり `blogs.category` の完全一致で記事を抽出する。

補足:
- `categories` に登録されていても、該当する公開記事が0件の場合は空のカテゴリ別ページを生成するか、リンクのみ出力してページ生成は行わないかを実装時に選択する
- 初期実装では、リンク切れを避けるため、カテゴリテーブルに登録されたカテゴリのカテゴリ別ページを生成する
- 空カテゴリページでは、記事がない旨を表示する

### 7.3 静的HTML再生成
カテゴリを作成、更新、削除した場合は、トップページを再生成対象とする。

再生成対象:
- `index.html`
- カテゴリ別記事一覧ページ
- `sitemap.xml`

補足:
- カテゴリ名変更時は、変更前カテゴリの公開ディレクトリが不要になるため削除対象とする
- `target=all` はカテゴリテーブルにもとづくトップページカテゴリ一覧を出力する
- `target=index` はカテゴリテーブルにもとづくトップページカテゴリ一覧を出力する

## 9. テスト観点
### 9.1 バックエンド
- カテゴリ一覧が `sort_order` 順で返却されること
- 親カテゴリのみのカテゴリを登録できること
- 親カテゴリ + 子カテゴリのカテゴリを登録できること
- 同一カテゴリ名の重複登録がエラーになること
- カテゴリ更新時に `updated_at` が更新されること
- カテゴリ削除時に対象レコードが削除されること
- 不正なカテゴリ名がバリデーションエラーになること

### 9.2 フロントエンド
- カテゴリ一覧が表示されること
- カテゴリを新規追加できること
- カテゴリを編集できること
- カテゴリを削除できること
- 記事編集画面でカテゴリ候補を選択できること
- 既存記事の未登録カテゴリが失われないこと

### 9.3 公開出力
- トップページのカテゴリ一覧が `categories` テーブルの順序で出力されること
- 公開済み記事が0件のカテゴリもトップページに表示されること
- 子カテゴリが親カテゴリ配下に表示されること
- カテゴリテーブルが空の場合に空状態表示になること
- カテゴリ別記事一覧ページのリンクが生成されること
- カテゴリ名変更後に不要なカテゴリディレクトリが削除されること

## 10. 将来拡張
- `blogs.category` 文字列ではなく `category_id` 参照へ移行する
- 親カテゴリと子カテゴリを別テーブルで階層管理する
- ドラッグアンドドロップによる並び替えを追加する
- カテゴリごとの説明文や表示色を管理する
- カテゴリ削除時に利用中記事を検出し、削除前に警告する
- カテゴリ別記事一覧ページのメタ情報をカテゴリごとに設定可能にする
